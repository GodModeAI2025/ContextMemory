Von der Datei zum Wissens-Node

Wie Context Memory aus 15+ Dateiformaten strukturiertes Wissen macht.

01
15+ Formate — Was reingeht

Du lädst eine Datei hoch und sagst Claude, dass er sie aufnehmen soll. Der Skill wählt automatisch die beste Extraktionsmethode: PDF→pypdf, Word→pandoc, Excel→pandas, PowerPoint→python-pptx, JSON→Strukturanalyse, Bilder→PIL-Metadaten.

📕
PDF
Text pro Seite. OCR-Fallback für Scans.
📝
Word
Headings = Abschnittsgrenzen.
📊
Excel
Spalten, Typen, Samples pro Sheet.
🎬
PowerPoint
Slide-Text + Speaker Notes.
🔧
JSON/CSV
Strukturanalyse, Schema, Statistiken.
💻
Code & mehr
Markdown, HTML, EPUB, YAML, Bilder…
02
Splitting — Wann was sinnvoll ist

Die Frage ist: Ein großes PDF als ein Node oder viele kleine? Das hängt vom Dokument ab.

single → Ein Node pro Datei. Für kurze Dokumente.
sections → Pro Heading. Für Specs und Handbücher.
pages → Pro PDF-Seite. Wenn keine Headings vorhanden.
slides → Pro Slide. Für Präsentationen.
sheets → Pro Excel-Sheet. Für thematisch getrennte Blätter.
03
Duplikaterkennung

Wenn du die gleiche Datei zweimal aufnimmst, warnt der Skill. Er vergleicht Titel und Tags und bietet dir an: trotzdem speichern, aktualisieren oder abbrechen.

Tipps für bessere Ergebnisse

Tags mitgeben macht dein Wissen durchsuchbar. Zeitbezug nennen verhindert Widersprüche. Relevanz angeben hilft der Suche. Und: verknüpfe sofort beim Aufnehmen — spart nachträgliches Verbinden.
