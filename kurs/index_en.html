01
Claude forgets everything. This skill changes that.

Normally, Claude starts every session from scratch. Your architecture decisions, bug fixes, research findings — all gone. Context Memory changes this fundamentally.

Just tell Claude what matters. The skill stores it in a Context TreeA hierarchical tree structure that organizes your project knowledge — by type, relevance, and relationships. — and next time, everything is there again.

── Without Context Memory ──
Session 1: "We're using PostgreSQL" → stored in Claude's head
Session 2: "What database do we use?" → ❌ No idea

── With Context Memory ──
Session 1: "Context Memory: Remember — we use PostgreSQL" → ✅ dec-001 saved
Session 2: "Context Memory: What database do we use?" → ✅ PostgreSQL (decision, critical)

05
Knowledge has an expiry date — and a certainty level

Not every piece of information is equally certain. And "Revenue 28B" and "Revenue 34B" aren't contradictions — they're a time seriesEach node can carry a source date, a validity window, and temporal confidence. This makes chronological changes visible instead of treating them as errors..

Temporal Dimension:
"AI adoption 12%" ← McKinsey, Q1 2025, confidence: explicit
"AI adoption 23%" ← Accenture, Q4 2025, confidence: explicit
→ Not a contradiction! Time series: +11 points in 9 months

Confidence:
[bug-002] Memory leak in order service
  Relevance: critical | Confidence: low
  → Important, but not confirmed yet. Once confirmed: confidence → high
What is the confidence rating for?
It determines how high a result ranks in search
It shows whether information is confirmed or still a hypothesis
It defines how long a knowledge block is stored
